# Sinitek AI VS Code CLI 插件功能清单

> 适用版本：`0.2.0`（基于当前仓库代码整理）

## 1. 插件入口与基础能力

- 在 VS Code Activity Bar 提供独立面板入口（内置 Webview 聊天界面）
- 提供状态栏入口，可一键打开聊天面板并显示当前 CLI
- 支持命令面板快速操作：选择 CLI、运行 CLI、打开面板、思考模式快捷运行
- 可配置启动后自动打开面板

## 2. 多 CLI 统一接入

- 支持 `codex` / `claude` / `gemini` 三种本地 CLI
- 每个 CLI 可独立配置可执行命令与默认参数
- 支持按 CLI 切换当前会话与执行目标
- 运行输出支持流式展示，错误信息在面板内反馈
- macOS 下可在工具设置中选择对话任务执行 shell（zsh / bash）
- 兼容跨平台命令解析（含 Windows 常见 npm 全局安装路径场景）
- 插件加载后预检测 `codex/claude/gemini` 可用性，切换到未安装 CLI 分组时弹出“一键安装”提示（确认后执行安装命令）

## 3. 执行模式（一次性 + 交互式）

- 一次性模式：每次发送提示词独立拉起进程执行
- 交互式模式（Beta）：`codex` / `claude` 支持常驻 Runner，复用会话上下文
- 交互模式支持 `coding` / `plan` 两种子模式
- 交互模式异常时自动降级到一次性模式
- 支持手动停止当前任务，停止时会尝试结束进程树

## 4. 会话与历史管理

- 按 CLI 维护独立会话列表（新建、切换、删除、清空）
- 支持并行任务会话 Tab：点击「+」新建并行会话，Tab 支持切换/关闭，关闭后按顺序自动重排；Tab 标题按分组独立计数（claude/codex/gemini 各自编号），序号 1 不显示（如 codex、codex2、claude、claude2），且同一组 Tab 可混用 codex/claude/gemini
- 每个 Tab 绑定独立会话：发送任务优先使用该 Tab 的 sessionId；不同 Tab 的流式输出、消息落盘与任务计时互不影响
- 任务执行中仍可新建并行会话 Tab，并可切换交互子模式（编码/规划，作用于后续任务）
- 支持不同 Tab 并发执行任务：切换到新 Tab 发送任务不会中断其它 Tab 正在执行的任务
- 任务执行 cwd 解析优先使用当前编辑文件所在工作区，其次使用工作区根目录；无工作区时回退到编辑文件目录并缓存最近有效 cwd，降低新建 Tab 时误落到用户主目录的概率
- 发送任务时携带 Tab 上下文（tabId + CLI），后台按目标 Tab 路由执行，避免因切换时序导致误中断其它任务
- 当仅有 1 个并行会话时不显示多 Tab 栏；Tab 机制与历史会话列表解耦
- 切换并行会话 Tab 时仅做显示切换，不重置任务计时；底部操作区状态（停止、队列、提示词、流式消息弹窗等）按 Tab 独立保持并随切换同步
- 多 Tab 同时执行时，切换 Tab 会按目标 Tab 重新加载对应会话消息；运行计时按 Tab 独立保持，不会串用其它 Tab 的计时
- 会话历史面板中的“重置当前会话”只重置当前 Tab，会清空当前 Tab 会话绑定并强制后续任务创建新会话 ID（不复用旧会话 ID）
- 顶部“新建会话（+）”右侧提供“重置当前会话”图标，点击后仅重置当前 Tab（不会影响其它 Tab）
- 自动生成并维护会话标签（含首条提示词信息）
- 历史会话“加载”支持跨 CLI 生效：当当前 Tab CLI 与目标会话 CLI 不一致时，会将当前 Tab 切换为目标 CLI 并加载对应历史消息
- 记录提示词历史，可回看与复用
- 会话、消息、提示词历史可持久化到本地目录 `~/.sinitek_cli/`
- 支持工作区维度保存偏好（当前 CLI、交互模式、思考模式等）

## 5. 提示词输入增强

- 自动上下文标签：`Current File` / `Selection`
- 发送时自动附加 `@相对路径` 与选区范围提示（不直接注入整段源码）
- 支持 `@` 路径选择器（文件/目录快速插入）
- 支持拖拽路径、上传附件、粘贴附件并保存到临时目录再引用
- 当任务执行中再次发送，支持“加入队列”或“暂停并发送”

## 6. 输出渲染与任务观测

- 聊天消息按角色展示（user/assistant/system/trace）
- 支持 trace 信息渲染（含工具调用、命令执行、文件变更等）
- Claude（one-shot / interactive）与 Codex/Gemini 统一走 Trace 分段事件渲染，运行时气泡风格一致
- Claude 交互模式支持解析 thinking / tool_use / tool_result 事件；其中 Bash 工具会映射为 `exec` 命令气泡，命令标签与 Codex 分组保持一致
- Gemini 分组 trace 输出对齐 Codex：支持将 `running/executing command`、`tool/tool result`、`[ERROR]/[WARN]` 规范化为类型气泡，并过滤 `.npmrc/YOLO/IDE companion` 启动噪声
- 气泡合并策略统一：命令执行、文件变更、工具调用/工具结果、错误/警告、网络检索默认独立气泡；连续普通状态文本默认合并到同一气泡
- 对“执行命令”trace 增加命令功能标签（如测试/构建/Python/Git 查询/检索/读取/写入等）
- 长脚本类命令（如 heredoc Python）默认折叠，可手动展开查看完整命令内容
- 文件变更 trace 默认折叠，支持展开查看完整变更；新增行绿色、删除行红色高亮
- `kind=thinking` 的运行中内容并入助手气泡流式展示，不单独生成 trace 气泡
- 新增气泡时仅在已贴底时自动滚动（贴底判定阈值 20px）；非贴底阅读时保持当前位置
- 非贴底状态会在右下角显示蓝色“回到底部”浮动图标，可一键跳转到最新消息
- 新增气泡不会重置既有气泡的展开/收起状态（如完整命令折叠）
- 支持任务列表（Task List）展示与更新
- 当前任务进入完成/失败/停止状态后，自动关闭并清空当前 Task List 面板，避免残留上一个任务列表
- 支持“查看当前/最近已完成任务提示词”与队列提示词查看、编辑并保存、顺序调整；任务结束后在“任务已完成,执行 xx:xx”同一行保留“提示词”按钮，重载会话后仍可回看
- 支持“流式消息”弹窗查看当前运行任务与最近已完成任务的原始流式输出（按条目单行记录，可点击展开完整内容），运行中自动刷新；当滚动条位于底部附近（50px）时自动跟随到底部，离开底部后不抢占滚动；任务结束后在“任务已完成,执行 xx:xx”同一行保留“流式消息”按钮，重载会话后仍可回看；支持一键导出全部流式消息为单个 TXT 文件（默认导出到 Downloads）
- 支持常用命令入口（如压缩上下文）

## 7. 思考模式与提示词增强

- 支持全局或分 CLI 的思考模式配置：`off/on/low/medium/high/xhigh`
- 支持分 CLI、分思考档位配置额外参数（thinkingArgs）
- 支持分 CLI、分档位配置提示词前后缀（thinkingPromptPrefix/Suffix）
- 支持分 CLI、分档位注入工作区附加文件内容（thinkingWorkspaceFiles）
- 提供命令快速以 Thinking On/Off 运行当前 CLI

## 8. 规则管理（Global / Project）

- 面板内可加载与编辑规则文件
- 支持规则作用域切换：全局规则 / 项目规则
- 支持一键保存覆盖到多个 CLI 目标
- 已适配规则文件约定：
  - Codex：`AGENTS.md`
  - Claude：`CLAUDE.md`
  - Gemini：`GEMINI.md`

## 9. 配置中心（Config Manager）

- 提供独立配置管理面板（增删改查、重命名、排序、激活）
- 支持分别管理 Claude/Codex/Gemini 的配置内容
- 支持导出配置到本地下载目录
- 支持配置备份与备份列表查看
- 支持 MCP Marketplace 列表读取；其中 Codex 平台通过 `codex mcp add` 命令安装，并通过 `codex mcp list --json` 回读已安装状态（不再仅依赖手改 TOML）
- 支持 Codex Skills 扫描与启用/禁用状态管理（同步到 `~/.codex/config.toml` 受控区块）

## 10. 国际化、日志与稳定性

- 插件 UI 支持中英文与 Auto（跟随 VS Code）
- Webview 运行异常可回退到错误提示页，便于定位问题
- 支持 debug 日志开关，日志写入本地目录
- 日志目录 `~/.sinitek_cli/logs` 自动保留最近 7 天，执行任务时会异步清理过期日志
- 单个日志文件最大 10MB，超过后自动按数字后缀分段（如 `.1.log`、`.2.log`）
- 工具设置项（交互开关/调试开关/语言/任务 Shell）会持久化到本地配置，重载后自动恢复
- 支持临时附件目录定时清理
- 开发模式下可按配置临时关闭 VS Code 扩展市场自动更新检查
