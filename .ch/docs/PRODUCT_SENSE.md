# ToB 产品思维基线

这个骨架是为复杂企业级系统准备的，因此默认从 ToB 视角看需求，而不是只看“功能是否能点出来”。

## 默认关注点

- 角色与权限：谁能看、谁能改、谁能批、谁能导出
- 可审计：关键动作必须能追溯到人、时间、上下文
- 可配置：组织、流程、字段、通知、策略通常需要配置能力
- 可运营：需要列表、筛选、批处理、回查、补偿、重试
- 可集成：导入导出、对接第三方、事件同步、主数据协同
- 可治理：错误定位、数据修复、风险隔离、灰度能力

## 写需求或拆任务时必须补齐的视角

- 主要使用角色是谁
- 关键成功路径是什么
- 异常路径和边界条件有哪些
- 成功后如何验证
- 是否影响权限、审计、报表、通知、批处理、集成

## 不建议的需求表达方式

- “做一个页面”
- “加一个字段”
- “支持导出”

这些描述都缺少业务目标、角色、约束和验收标准，无法直接支持高质量 agent 执行。
