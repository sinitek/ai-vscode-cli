# 核心信念

这些信念不是宣传语，而是为了让仓库长期对代理和人类都保持可读、可维护、可演化。

## 1. 仓库知识就是系统事实来源

如果信息不在仓库里，代理基本等价于看不见它。

## 2. `AGENTS.md` 是目录，不是百科

根级 `AGENTS.md` 只承载稳定规则和导航，深层知识放进 `.ch/docs/` 或更近的局部规则文件。

## 3. 非平凡任务必须留下执行痕迹

计划、验收标准、验证结论、关键决策都应该进入仓库，而不是留在一次对话里。

## 4. 最小但高价值的工具优于工具堆积

skills 和 MCP 过多会制造噪音，只有能显著提升复杂任务完成率的能力才值得常驻。

## 5. 优先无魔法、强约定、可检索的实现

代理擅长在明确边界和稳定模式下高速工作，不擅长长期维护一堆隐式规则和例外。

## 6. 共享规则优于复制粘贴

稳定的公共契约、公共工具、公共组件应被复用，而不是在不同角落重复实现。

## 7. 边界必须先校验再使用

外部接口、配置、事件、表单、消息、文件导入都不能假定形状总是正确。

## 8. 文档要分层和渐进式暴露

索引页负责导航，主题页负责规则，生成页负责清单。不要让任何单个文档失控。

## 9. 验证要尽量自动化，但先从最小可行做起

先有针对性验证，再演进成更完整的自动化闭环。

## 10. 技术债要持续回收，而不是定期爆炸

质量治理、文档修剪、规则收敛应该变成持续动作，而不是季度运动式大扫除。
