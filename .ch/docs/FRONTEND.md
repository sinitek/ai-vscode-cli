# 前端与体验层基线

当前仓库还未绑定具体前端技术栈，但未来无论选什么框架，都遵守下面的 AI 友好约束。

## 核心原则

- 优先设计系统和语义化 token，不写随意颜色或零散视觉规则。
- 页面、路由、组件、状态、接口要按业务域组织，不按“看起来方便”堆在一起。
- 体验层负责流程编排和交互表达，不复制领域规则。
- 可访问性、空态、错误态、加载态、权限态必须是默认考虑项。

## 对 ToB 场景尤其重要的点

- 角色差异明显：管理员、运营、审核、客服、财务、实施、审计等路径不同。
- 列表、筛选、批量操作、导入导出、审批流、审计可追踪是常态，不是边角需求。
- 复杂表单和复杂工作流需要先定义状态机或步骤图，再谈页面细节。

## 实施约束

- 如果未来项目引入主题系统，所有样式必须优先使用语义化主题能力。
- 前端不得绕过明确契约直接依赖数据库或未定义的后端内部结构。
- 每个前端子应用都应有自己的局部 `AGENTS.md` 和页面地图。

## 验证思路

- 关键用户旅程要可自动化回归。
- 可视回归、截图、DOM 快照、控制台错误检查应逐步纳入能力建设。
