# 安全基线

这个仓库的安全策略强调：**最小暴露、边界校验、最小副作用、最少外部依赖。**

## 当前基线

- 不把秘密信息写进仓库。
- skills 和 MCP 只保留少数高价值项。
- 仓库当前不提供项目级 shell 审批/沙箱护栏，执行边界由启动参数和外层运行环境决定。
- 高副作用命令应依赖外层隔离环境、人工流程或 CI 权限设计，而不是仓库内规则文件。
- 项目知识优先本地化，减少对外部不可控上下文的依赖。

## 未来真实项目必须落地的点

- 输入边界校验与输出脱敏
- 权限模型和角色矩阵
- 关键变更的审计日志
- 秘密管理与环境隔离
- 第三方依赖治理与升级策略
- 安全事件 runbook

## 对代理尤其重要的规则

- 不基于猜测构造安全关键逻辑。
- 不绕过权限边界临时“先跑通”。
- 不把生产配置、密钥、客户数据当测试夹具。

## 与 harness 的关系

安全不是后置审查动作，而是仓库结构、配置、命令规则、文档习惯共同形成的约束系统。
